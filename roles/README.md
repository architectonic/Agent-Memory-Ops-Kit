# Roles

Roles define bounded responsibilities for agents working with a project memory system.

A role is not a personality. It is an operating boundary.

## Role File Pattern

Each role should define:

- scope;
- responsibilities;
- non-responsibilities;
- useful skills;
- handoff expectations.

## Available Roles

- `architect.md`
- `developer.md`
- `researcher.md`
- `reviewer.md`
- `operator.md`
- `maintainer.md`

## Rule

Keep roles short. Use skills for procedures and runbooks for multi-step workflows.
