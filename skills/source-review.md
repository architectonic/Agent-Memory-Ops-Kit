# Source Review Skill

## Trigger

Use when asked to understand docs, code, tutorials, specifications, transcripts, issues, or existing repository material.

## Inputs

- Source artifact.
- User-stated goal.
- Relevant constraints.

## Procedure

1. Read the source before forming conclusions.
2. Identify native terminology.
3. Separate facts, decisions, assumptions, and open questions.
4. Distinguish source claims from inference.
5. Identify what evidence is missing.
6. Report findings before proposing changes.

## Verification

- Cite files, lines, commits, issues, or docs where possible.
- State when a conclusion is inferred rather than directly sourced.

## Failure Modes

- Skimming and guessing intent.
- Replacing source ontology with familiar patterns.
- Treating generated summaries as primary evidence.
- Making recommendations before understanding the source.
