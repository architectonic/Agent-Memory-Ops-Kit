# Docs Indexing Skill

## Trigger

Use when adding documentation references for tools, frameworks, services, or agent ecosystems.

## Inputs

- Tool or technology name.
- Official docs, if known.
- Reason the docs matter.
- Review date.

## Procedure

1. Prefer official documentation.
2. Record why the docs matter to the project.
3. Mark volatile information as review-needed.
4. Avoid adding long lists of weak links.
5. Keep private workspace links out of public repos.
6. Re-check current docs before making precise claims.

## Verification

The index should help a future agent find the right source without becoming a stale link dump.

## Failure Modes

- Adding unofficial links as source of truth.
- Treating current tool behavior as permanent.
- Including private or authenticated links.
- Creating an encyclopedia instead of an index.
