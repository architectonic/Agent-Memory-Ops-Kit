# Memory Maintenance Skill

## Trigger

Use when project memory, docs, logs, or agent notes have accumulated enough material to need pruning or consolidation.

## Inputs

- Existing memory files.
- Recent handoffs.
- Decision log.
- Goals.
- Changed files or release notes.

## Procedure

1. Identify durable lessons.
2. Identify stale, duplicated, or low-value notes.
3. Promote only source-backed or verified lessons.
4. Delete or archive junk.
5. Consolidate repeated guidance.
6. Preserve source links where possible.
7. Write a short maintenance report.

## Verification

The memory system should become easier for a fresh agent to read and act on.

## Failure Modes

- Accumulating notes without pruning.
- Turning temporary context into doctrine.
- Deleting decisions that still matter.
- Reorganizing files without reducing confusion.
