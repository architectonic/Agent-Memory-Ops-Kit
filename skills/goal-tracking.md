# Goal Tracking Skill

## Trigger

Use when a project has long-term goals that should guide future agent work.

## Inputs

- Stated goal.
- Scope.
- Completion evidence.
- Constraints.
- Current status.

## Procedure

1. Confirm the goal is explicitly stated.
2. Separate the desired state from implementation tasks.
3. Define evidence of completion.
4. Record constraints and non-scope.
5. Link related decisions or handoffs.
6. Set a review cadence.
7. Mark status clearly.

## Verification

A future agent should be able to tell whether current work supports, ignores, or conflicts with the goal.

## Failure Modes

- Inferring goals from vague conversation.
- Treating a goal as a task list.
- Keeping stale goals active.
- Recording goals with no completion evidence.
