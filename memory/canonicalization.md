# Canonicalization

Canonicalization is the process of converting useful information into durable project knowledge.

## Principle

Not every observation deserves permanent storage.

## Promote

Promote:

- verified facts;
- durable rules;
- stable constraints;
- architectural decisions;
- recurring failure corrections;
- validated terminology.

## Do Not Promote

- moods;
- temporary plans;
- speculation;
- one-off conversations;
- unsupported assumptions;
- stale implementation details.

## Procedure

1. Classify the information.
2. Verify the source.
3. Remove temporary context.
4. Store the smallest durable version.
5. Link to the source when possible.

## Test

If the information would still be useful six months from now, it is a candidate for canonical memory.
