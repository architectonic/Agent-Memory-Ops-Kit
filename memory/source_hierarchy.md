# Source Hierarchy

This file defines how agents should resolve conflicting information.

## Order Of Trust

1. Current user instruction.
2. Repository source artifacts.
3. Repository operating documents.
4. Cited external sources.
5. Derived summaries.
6. Inference.
7. Speculation.

Higher levels override lower levels.

## Important Rule

Memory is context, not evidence.

A memory file may help locate knowledge, but it does not automatically prove a claim.

## Conflict Resolution

When two sources disagree:

1. Prefer the more primary source.
2. Prefer the more recent source when authority is equal.
3. Label unresolved conflicts explicitly.
4. Do not silently merge contradictions.

## Anti-Pattern

Do not use hidden model memory, assumptions, prior chats, or intuition as authoritative project facts.
