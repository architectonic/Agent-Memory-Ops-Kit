# Dream Cycle Model

Use this model for periodic knowledge maintenance.

"Dream" means a structured reflection cycle: consolidate useful knowledge, reorganize messy files, remove junk, preserve purpose, and identify what should be improved next.

It is not autonomous self-direction, hidden agency, or mystical framing. It is repo hygiene for agent memory systems.

## Trigger

Run after a major work session, release, incident, or period of accumulated context.

## Procedure

1. Review recent handoffs, decisions, goals, and changed files.
2. Identify repeated mistakes or recurring patterns.
3. Promote only durable lessons into memory.
4. Remove stale, duplicated, or low-value notes.
5. Reorganize files only when it reduces future confusion.
6. Check whether current goals still matter.
7. Produce a short maintenance report.

## Maintenance Report Template

```md
# Dream Cycle Report

Date:
Scope reviewed:
Useful lessons promoted:
Junk removed:
Files reorganized:
Goals reviewed:
Open questions:
Next maintenance action:
```

## Rules

- Do not invent new purpose.
- Do not promote temporary mood or conversation tone into memory.
- Do not expand structure unless it clearly helps future work.
- Preserve source hierarchy.
- Prefer deletion over accumulation when notes no longer help.
