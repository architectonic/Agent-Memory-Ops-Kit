# Ontology Model

Use this model to define the native concepts of a project.

Ontology is the vocabulary and relationship map of the system. Agents should learn it from source artifacts instead of imposing familiar patterns.

## Template

```md
# Project Ontology

## Concepts

### Concept Name

Definition:
Source:
Related concepts:
Do not confuse with:

## Boundaries

What belongs inside the system:

What belongs outside the system:

## Relationships

- Concept A relates to Concept B because...

## Open Questions

- Question 1
```

## Rules

- Define concepts from source artifacts or human confirmation.
- Do not rename domain concepts casually.
- Keep assumptions labeled.
- Separate glossary terms from decisions.
