# Goal Model

Use this model to track long-term goals without confusing them with tasks.

A goal is a desired state. A task is a step toward that state.

## Goal Template

```md
# Goal Name

## Desired State

What should be true when this goal is achieved?

## Scope

What does this goal include?

## Non-Scope

What does this goal not include?

## Evidence Of Completion

- Observable evidence 1
- Observable evidence 2

## Constraints

- Constraint 1
- Constraint 2

## Current Status

proposed | active | paused | achieved | abandoned

## Related Decisions

- Link or reference relevant decisions.

## Review Cadence

When should this goal be reviewed?
```

## Goal Rules

- Do not infer goals from conversation context.
- Do not treat a goal as an implementation plan.
- Do not let stale goals control current work.
- Every goal needs evidence of completion.
- Long-term goals should be reviewed periodically.
