# AGENTS.md

Instructions for AI agents working in this repository.

## Mission

Preserve useful public patterns for agentic software work without importing private context.

This repo is a starter memory kit. Keep it generic, reusable, and safe to publish.

## Source Hierarchy

1. Current user instruction.
2. Repository files.
3. Cited public sources.
4. Inference.
5. Speculation.

Do not treat hidden memory, prior private conversations, or private repos as publishable source material.

## Public-Safety Boundary

Never add:

- secrets;
- credentials;
- private names;
- emails;
- handles;
- client-sensitive context;
- raw private transcripts;
- proprietary repo details;
- private source indexes.

Use placeholder names and generic examples.

## Operating Rules

- Read before writing.
- Prefer small, practical Markdown files.
- Keep examples generic.
- Separate reusable pattern from private context.
- Do not expand structure without a clear reader and use case.
- Do not add doctrine that belongs in a private memory vault.
- Make templates easy to copy into another repo.
- Verify any external-system mutation by reading back the resulting state.

## Completion Standard

A change is complete only when:

1. the affected files are readable;
2. examples contain no private identifiers;
3. instructions are actionable;
4. the repo remains useful to a fresh agent entering a new codebase.
